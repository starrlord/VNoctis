Getting started

From zero to
playing in minutes

Step 01
Configure & Launch

Set VNM_ROOT and VNM_ADMIN_PASSWORD in your .env file. All sub-paths derive automatically. Then run a single compose command — the Ren'Py SDK downloads itself on first boot.

01
.env
1# Required
2VNM_ROOT=/mnt/user/appdata/vnm
3VNM_ADMIN_PASSWORD=your-password
4# Optional
5HOST_PORT=6773
6RENPY_SDK_VERSION=8.5.2

Step 02
Auto-discover & Enrich

VNoctis scans your games directory and fuzzy-matches each title against VNDB. Cover art, synopses, ratings, screenshots, and tags are fetched and cached automatically. No manual entry needed.

Step 03
Build & Play

Click Build on any game card. The builder compiles a WebAssembly web distribution, compresses images up to 60% smaller, generates progressive download rules, and serves it in the in-browser player — all without leaving the UI.

03
Progressive download rules
1# GUI stays bundled
2- image game/gui/**
3# Backgrounds stream
4+ image game/**
5+ music game/audio/**
6+ voice game/voice/**
Technology

Built on a
modern foundation

⚛️
React 19 + Vite 6
Frontend · vnm-ui
Tailwind CSS v4 for styling, React hooks for state, Nginx for static file serving and API proxying.
⚡
Fastify 5
API Server · vnm-api
Node.js 24, Prisma ORM, SQLite — fast schema-driven REST with SSE log streaming and JWT auth.
🐍
FastAPI
Build Worker · vnm-builder
Python 3.11, Pillow, pngquant, jpegoptim — async build queue with concurrent job control.
🗃️
SQLite + Prisma
Database
Zero-dependency embedded DB with typed Prisma schema, migrations, and persistent named volumes.
🐳
Docker Compose
Containerisation
Health checks, resource limits, network isolation, named volumes. One command to rule them all.
📡
VNDB API
Metadata Source
Kana API with configurable rate limiting, fuzzy match threshold, and optional API token for higher limits.
Quick start

Up and running
in four commands

bash — vnoctis-manager
# 1. Clone the repository
$ git clone https://github.com/starrlord/VNoctis && cd VNoctis

# 2. Configure environment (set VNM_ROOT + VNM_ADMIN_PASSWORD)
$ cp .env.example .env && nano .env

# 3. Launch (SDK auto-downloads on first boot)
$ docker compose up -d

Creating network "vn-manager_default"
Creating vnm-api ... done
Creating vnm-builder ... done
Creating vnm-ui ... done

# 4. Open in browser
$ open http://localhost:6773
